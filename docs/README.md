# Documentation Index

Welcome to the Hypercaller documentation. This directory contains comprehensive documentation for the project.

## Core Documentation

### [Getting Started](./getting-started.md)
Complete setup guide including installation, AWS configuration, database setup, and running the application. **Start here if you're new to the project.**

### [Environment Setup Guide](./env-setup-guide.md)
Comprehensive guide for configuring environment variables including AWS credentials, Bedrock configuration, and Redis setup. Essential for local development.

### [Architecture Documentation](./architecture.md)
Project structure, architecture layers, data flow, security architecture, and technology stack overview. Understand how the system is designed.

### [API Reference](./api-reference.md)
Comprehensive documentation of all API endpoints including request/response formats, error handling, and security considerations.

### [Components Documentation](./components.md)
Complete documentation of all React components including props, usage examples, features, and implementation details.

### [Database Documentation](./database.md)
Detailed database schema, table structures, indexes, TTL configuration, GSI setup, and setup instructions.

### [Development Guide](./development.md)
Coding standards, best practices, testing guidelines, error handling, and contribution guidelines.

### [Testing Guide](./testing-guide.md)
Comprehensive testing guide for search functionality, covering end-to-end, performance, UI/UX, and integration testing.

### [Troubleshooting Guide](./troubleshooting.md)
Common issues and solutions for DynamoDB, Bedrock, Redis, rate limiting, caching, normalization, search issues, and general debugging tips.

### [Development Journey](./development-journey.md)
Comprehensive documentation of all major challenges encountered during development, their root causes, solutions implemented, and lessons learned. Essential reading for understanding the system's evolution.
